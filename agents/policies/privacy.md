# Política de Privacidade e Proteção de Dados - Fiz-Go

## Lei Geral de Proteção de Dados (LGPD)

### Princípios Fundamentais

A plataforma Fiz-Go adere rigorosamente aos princípios estabelecidos pela LGPD (Lei 13.709/2018):

1. **Finalidade**: Coleta de dados apenas para propósitos específicos e legítimos
2. **Adequação**: Compatibilidade com as finalidades informadas ao titular
3. **Necessidade**: Limitação ao mínimo necessário para realização das finalidades
4. **Livre Acesso**: Garantia de consulta facilitada sobre dados pessoais
5. **Qualidade dos Dados**: Exatidão, clareza e atualização dos dados
6. **Transparência**: Informações claras sobre tratamento de dados
7. **Segurança**: Medidas técnicas e administrativas de proteção
8. **Prevenção**: Adoção de medidas para prevenir danos
9. **Não Discriminação**: Vedação de tratamento para fins discriminatórios
10. **Responsabilização**: Demonstração de adoção de medidas eficazes

### Dados Coletados

#### Dados Pessoais Básicos
- Nome completo
- E-mail
- Telefone
- CPF (para profissionais)
- Data de nascimento
- Endereço e localização

#### Dados de Perfil Profissional
- Categoria de serviços
- Experiência profissional
- Portfólio de trabalhos
- Avaliações e comentários
- Disponibilidade de horários

#### Dados de Uso da Plataforma
- Histórico de navegação
- Preferências de serviços
- Interações via chat
- Dados de geolocalização (quando autorizado)
- Informações de dispositivo (logs técnicos)

#### Dados Financeiros
- Dados para pagamento (tokenizados)
- Histórico de transações
- Dados bancários (criptografados)

### Bases Legais para Tratamento

1. **Consentimento**: Para dados não essenciais ao serviço
2. **Execução de Contrato**: Para prestação dos serviços da plataforma
3. **Legítimo Interesse**: Para melhorias do serviço e segurança
4. **Cumprimento de Obrigação Legal**: Para requisitos fiscais e regulatórios

### Direitos dos Titulares

Os usuários têm direito a:

1. **Confirmação e Acesso**: Saber se seus dados são tratados
2. **Correção**: Corrigir dados incompletos ou inexatos
3. **Anonimização/Bloqueio**: Solicitar anonimização ou bloqueio
4. **Eliminação**: Solicitar exclusão de dados desnecessários
5. **Portabilidade**: Receber dados em formato interoperável
6. **Revogação de Consentimento**: Retirar consentimento a qualquer tempo
7. **Informação sobre Compartilhamento**: Saber com quem dados são compartilhados
8. **Oposição**: Opor-se ao tratamento quando baseado em legítimo interesse

### Implementação Técnica

#### Medidas de Segurança
- Criptografia de dados sensíveis (AES-256)
- Tokenização de dados de pagamento
- Autenticação multifator para acessos administrativos
- Logs de auditoria com monitoramento contínuo
- Backups seguros com retenção definida

#### Privacy by Design
- Minimização de dados desde a concepção
- Configurações de privacidade como padrão
- Controles granulares de consentimento
- Anonimização automática de dados desnecessários

#### Gestão de Consentimento
- Interface clara para consentimentos específicos
- Registro detalhado de consentimentos
- Facilidade para revogação
- Notificação sobre mudanças nas políticas

### Compartilhamento de Dados

#### Terceiros Autorizados
- Processadores de pagamento (com tokenização)
- Serviços de geolocalização (dados anonimizados)
- Provedores de infraestrutura em nuvem (com criptografia)
- Serviços de autenticação social (dados mínimos necessários)

#### Transferência Internacional
- Adequação do país de destino
- Cláusulas contratuais padrão quando necessário
- Garantias adequadas de proteção

### Retenção de Dados

#### Períodos de Retenção
- Dados de cadastro: Durante vigência da conta + 5 anos
- Dados transacionais: 5 anos (requisito fiscal)
- Logs de acesso: 12 meses
- Dados de marketing: Até revogação do consentimento

#### Exclusão Automática
- Sistema automatizado de exclusão por prazo
- Notificação prévia ao usuário sobre exclusão
- Manutenção apenas de dados legalmente obrigatórios

### Incidentes de Segurança

#### Processo de Resposta
1. **Detecção**: Monitoramento contínuo e alertas automáticos
2. **Contenção**: Isolamento imediato de sistemas afetados
3. **Avaliação**: Análise de impacto e dados envolvidos
4. **Notificação**: ANPD em até 72h, usuários quando aplicável
5. **Remediação**: Correção de vulnerabilidades
6. **Documentação**: Registro completo do incidente
